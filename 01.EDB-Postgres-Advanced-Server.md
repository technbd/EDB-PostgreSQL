## EDB Postgres Advanced Server (EPAS):

EDB Postgres Advanced Server (EPAS) is EnterpriseDB‚Äôs commercial, enterprise-grade PostgreSQL distribution. It is a proprietary-enhanced version built directly on top of open-source PostgreSQL. It is fully PostgreSQL-compatible, but adds Oracle compatibility, enterprise tooling, and commercial support. 


### EPAS Architecture (Same Core as PostgreSQL):

_Internally, EPAS still uses:_
- PostgreSQL storage engine
- MVCC
- WAL
- Streaming replication
- Logical replication
- Same `psql`, `pg_dump`, `pg_basebackup`


### Why Enterprises Choose EPAS:

While it includes everything from standard PostgreSQL (you get all community features + performance improvements), EPAS adds many **enterprise-oriented extensions**, especially focused on:

#### 1. Oracle¬Æ Database Compatibility ‚Äî the #1 selling point:
Very high compatibility with Oracle PL/SQL, packages, data types, built-in functions, hierarchical queries, DECODE, CONNECT BY, Oracle-style catalog views, MERGE enhancements, and more. This dramatically reduces migration effort/cost from Oracle ‚Üí Postgres (often 80-95% of Oracle code can run with minimal or no changes).

- Minimal application rewrite
- Run Oracle-like schemas and code
- Lower licensing cost


#### 2. Advanced Security Features:
- Transparent Data Encryption (TDE)
- Data redaction (masking sensitive data)
- Enhanced auditing
- SQL/Protect (protection against SQL injection)
- Advanced row-level security & fine-grained access controls
- Better support for regulatory compliance (FIPS, etc.)


#### 3. Performance & Monitoring Enhancements: 
- Query optimizer hints
- Advanced performance tuning & diagnostics tools
- Better handling of large-scale workloads
- EDB-specific extensions for workload analysis


#### 4. Developer & DBA Productivity Tools:
- EDBPlus (Oracle SQLPlus-like client)
- EDB*Loader (enhanced bulk loader)
- Compatible procedural logic & packages
- Additional utilities for application development


#### 5. Other Enterprise Capabilities:
- Advanced replication features (especially from version 14+)
- Better tools for high-availability & disaster recovery integration


#### 6. Mission-Critical Databases
- Banking
- Telecom
- Government
- ERP systems



### EPAS vs EDB Postgres Extended (Clarification):

_EDB has two main enterprise offerings:_

| Product                   | Purpose                                                |
| ------------------------- | ------------------------------------------------------ |
| **EPAS**                  | Oracle-compatible PostgreSQL                           |
| **EDB Postgres Extended** | PostgreSQL + performance & security (no Oracle syntax) |

| Feature Area | Community PostgreSQL | EDB Postgres Advanced Server (EPAS) |

| Base engine  | ‚úì Full open source | ‚úì Same + enterprise extensions |

### EPAS Licensing Model:
- **Software installs freely**
- **Enterprise features require a license**
- Without license:
    - Acts like enhanced PostgreSQL
    - Oracle compatibility still works
- With license:
    - Advanced tooling
    - Security packs
    - Performance features
    - Official support

## Install EDB Postgres Advanced Server:


### Prerequisites:

- `root` or `sudo` privileges.
- EDB Account: Credentials for the EDB repositories to download software.
- Internet access for repository setup and package download.



### Install Required System Packages:

```
dnf -y install epel-release
dnf -y install yum-utils
```


```
dnf repolist all | grep enterprisedb
```


_If enable or disable repo:_
```
dnf config-manager --enable enterprisedb*
dnf config-manager --disable enterprisedb*
```


_Search available packages:_
```
dnf search edb-as*
```


### Install EPAS:

Choose the version of EPAS you want (e.g., 14, 15, 16, etc.). The package names follow this pattern:

```
edb-as<version>-server
edb-as<version>-server-client
```


_Installing **EDB Postgres Advanced Server** v16:_

```
dnf install -y edb-as16-server edb-as16-server-client edb-as16-server-contrib 
```



_Installing EDB Postgres Enterprise Manager:_
```
dnf install -y edb-pem 

dnf install -y edb-pem-agent 
```



_Installing EDB Postgres Failover Manager:_
```
dnf install -y edb-efm50

dnf install -y edb-efm52
```


_Installing EDB Postgres Replication Server:_
```
dnf install -y edb-xdb
```



_Installing EDB Pgpool-II:_
```
dnf install -y edb-pgpool44 edb-as16-pgpool44-extensions
```


_Installing EDB pgBouncer:_
```
dnf install -y edb-pgbouncer123
```


_Installing etcd and Depending on your system, enable the following repositories:

- RHEL 9: pgdg-rhel9-extras
- RHEL 8: pgdg-rhel8-extras
- RHEL 7: pgdg-rhel7-extras

```
dnf --enablerepo=pgdg-rhel8-extras install -y etcd
```


_Installing Patroni:_
```
dnf install -y edb-patroni
```


_Installing barman:_
```
dnf -y install barman
```




### Initialize Database:

_Initial configuration:_
```
/usr/edb/as16/bin/edb-as-16-setup initdb
```


#### Configure `pg_hba.conf`:

```conf

# IPv4 local connections:
host    all             all             0.0.0.0/0            md5
```


#### Start the Database:

_Start EPAS using `systemd` service (e.g., `edb-as-14` or `edb-as-16`). You may be able to manage EPAS like this:_

```
systemctl start edb-as-16
systemctl status edb-as-16
```


```
netstat -tlpn | grep edb

tcp        0      0 0.0.0.0:5444        0.0.0.0:*      LISTEN      96275/edb-postgres
```



### Connect to Database::

_User `enterprisedb` (instead of `postgres`):_
```
su - enterprisedb
```


```
psql edb
psql -d edb
psql -U enterprisedb -d edb
```


```
ALTER ROLE enterprisedb IDENTIFIED BY 'password';
```
